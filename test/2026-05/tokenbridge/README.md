## Root cause analysis

- **Title**: TokenBridge accepted a Wormhole transfer VM as mint authority and minted unbacked ALPH
- **Severity**: `critical`
- **Confidence**: `medium`
- **Violated invariant**: Wrapped-token minting must be authorized only by a unique, correctly signed Wormhole transfer message from a registered foreign TokenBridge emitter for the target chain and amount.

### Final root cause

TokenBridge `completeTransfer(bytes)` reaches `Bridge._completeTransfer`, accepts the supplied encoded Wormhole VM through `wormhole().parseAndVerifyVM(encodedVm)` and `verifyBridgeVM(vm)`, and enters the foreign-token branch that mints wrapped ALPH to the attacker. ALPH totalSupply and the attacker balance increase by the same amount, proving a mint entitlement rather than a transfer-only drain. The exact internal Wormhole validation condition that returned `valid` cannot be pinpointed because the Wormhole proxy implementation source is absent from the supplied artifacts, so the RCA is partial.

### Recommended fixes

- Ensure `Bridge._completeTransfer` mints only after Wormhole VM validation proves guardian quorum, registered emitter, target chain, replay uniqueness, and authorized amount.
- Patch and audit Wormhole implementation `0x092d8e03f672300359e035ba174a922ed414123e` so `parseAndVerifyVM` cannot return valid for forged, malformed, under-threshold, or otherwise unauthorized transfer VAAs.
- Add invariant tests that ALPH totalSupply can increase only by a transfer VAA that passes independently verified guardian-set and emitter checks.
